# Лабораторная работа №2 – Коллекция объектов

## Цель работы

Научиться работать с коллекциями объектов, реализовать собственный контейнерный класс, освоить итерацию и базовые операции управления коллекцией.

## Предметная область

Банковские счета (класс `BankAccount` из ЛР-1).

## Реализованный контейнер

**Класс `BankAccountCollection`** – коллекция для хранения объектов `BankAccount`.  
Внутреннее хранилище: `self._items = []`.

### Возможности 

- **Добавление** с проверкой типа и уникальности (по номеру счёта `_number`).
- **Удаление** по объекту и по индексу.
- **Получение всех элементов** (копия списка).
- **Поиск** по владельцу (`find_by_owner`) и по номеру счёта (`find_by_number`).
- **Магические методы**:
  - `__len__` – длина коллекции.
  - `__iter__` – итерация в цикле `for`.
  - `__getitem__` – доступ по индексу `collection[i]`.
- **Сортировка**:
  - Универсальный `sort(key, reverse)`.
  - `sort_by_balance` и `sort_by_owner` для удобства.
- **Фильтрация**:
  - `get_active` – возвращает новую коллекцию активных счетов.
  - `filter(predicate)` – универсальный метод с произвольным условием.
- **Контроль дубликатов** – нельзя добавить два счета с одинаковым номером.

## Демонстрация (`demo.py`)

В `demo.py` реализованы следующие сценарии:

1. Создание нескольких счетов и добавление их в коллекцию.
2. Вывод длины коллекции и перебор через `for`.
3. Поиск по номеру и по имени владельца.
4. Удаление по объекту и по индексу.
5. Сортировка по балансу и по владельцу (в т.ч. обратный порядок).
6. Фильтрация активных счетов и счетов с балансом > 500.
7. Попытка добавить дубликат – перехват исключения.
8. Доступ по индексам `[0]`, `[-1]`.
