## Цель работы

Освоить механизм наследования классов, научиться строить иерархию объектов, переиспользовать код, переопределять методы и реализовывать полиморфное поведение.

## Предметная область

Банковские счета (на основе класса `BankAccount` из ЛР-1).

## Реализованная иерархия

- **Базовый класс** (`base.py`): `BankAccount` – общий банковский счёт.
- **Производные классы** (`models.py`):
  - `CreditAccount` – кредитный счёт.
  - `SaveAccount` – сберегательный счёт.

### Базовый класс `BankAccount`

Содержит общие атрибуты:
- `owner` – владелец (строка)
- `_number` – номер счёта (строка, 8–16 цифр)
- `balance` – баланс (число, неотрицательный для обычного счёта)
- `_currency` – валюта (`rub` или `dollar`)
- `is_active` – активен ли счёт (bool)

Методы:
- `top_balance(amount)` – пополнение
- `transfer_balance(amount)` – списание
- `apply_monthly_interest()` – заглушка (выбрасывает `NotImplementedError` или выводит сообщение)
- `__str__`, `__repr__`, `__eq__` (сравнение по номеру счёта)
- свойство `currency` с проверками

### Производные классы

#### `CreditAccount` (кредитный счёт)

**Добавленные атрибуты:**
- `lim` – кредитный лимит
- `percentages` – процентная ставка (например, 15%)
- `min_count` – минимальный ежемесячный платёж

**Переопределённые методы:**
- `__str__` – добавляет `", ставка={self.percentages}%"`
- `top_balance(amount)` – пополнение с комиссией 15% (зачисляется `amount - amount*0.15`)
- `transfer_balance(amount)` – списание с дополнительной комиссией 15% (списывается `amount + amount*0.15`)
- `apply_monthly_interest()` – если баланс отрицательный, начисляет проценты на сумму долга (увеличивает долг)

#### `SaveAccount` (сберегательный счёт)

**Добавленные атрибуты:**
- `percentages` – процентная ставка (например, 5%)

**Переопределённые методы:**
- `__str__` – добавляет `", ставка={self.percentages}%"`
- `top_balance(amount)` – пополнение с комиссией 5% (зачисляется `amount - amount*0.05`)
- `transfer_balance(amount)` – списание с дополнительной комиссией 5% (списывается `amount + amount*0.05`)
- `apply_monthly_interest()` – начисляет проценты на положительный баланс (увеличивает баланс)

## Коллекция (`collection.py`)

Класс `BankAccountCollection` из ЛР-2 адаптирован для работы с любыми наследниками `BankAccount` благодаря использованию `isinstance(account, BankAccount)` в методе `add`.

**Дополнительные методы для ЛР-3:**
- `filter_by_type(class_type)` – возвращает новую коллекцию с объектами указанного типа (включая наследников).
- `get_credit_accounts()` – возвращает только кредитные счета.
- `get_savings_accounts()` – возвращает только сберегательные счета.

Остальные методы (`add`, `remove`, `remove_at`, `find_by_owner`, `find_by_number`, `sort`, `filter`, `get_active` и магические методы) остались без изменений.

## Демонстрация (`demo.py`)

`demo.py` показывает следующие сценарии (оценка 5):

1. **Создание объектов** – обычный счёт (`BankAccount`), кредитный (`CreditAccount`), сберегательный (`SaveAccount`).
2. **Добавление в коллекцию** – проверка работы `add`, `__len__`, `__iter__`.
3. **Полиморфизм пополнения/списания** – для каждого счёта вызываются `top_balance(1000)` и `transfer_balance(200)`. Выводятся балансы после каждой операции. Комиссии отличаются в зависимости от типа счёта.
4. **Ежемесячные операции** – вызов `apply_monthly_interest()` для всех счетов в коллекции. Результат разный:
   - Кредитный счёт: начисление процентов на отрицательный остаток.
   - Сберегательный счёт: начисление процентов на положительный остаток.
   - Обычный счёт: сообщение, что начисление не предусмотрено (или ничего не делает).
5. **Проверка типов** – цикл по коллекции с использованием `isinstance()` для определения типа каждого счёта.
6. **Фильтрация по типу** – получение только кредитных и только сберегательных счетов через `filter(lambda a: isinstance(a, CreditAccount))`.
7. **Сортировка** – сортировка смешанной коллекции по балансу (работает благодаря общему атрибуту `balance`).
8. **Поиск по номеру** – демонстрация `find_by_number`.
9. **Удаление и дубликаты** – удаление обычного счёта и попытка добавить счёт с уже существующим номером (перехват исключения).
